# Урок: Как создать HTML-презентацию быстро и без лишних правок

Этот урок написан на основе реального проекта (конференция Mailfit/Letteros, апрель 2026).
Следуй этому порядку — и следующую презентацию сделаешь за 1 сессию вместо 10.

---

## Шаг 0. Подготовь материалы ДО начала работы

Прежде чем открывать редактор — собери всё в одну папку:

```
conf/
├── index.html          ← возьми template.html как основу
├── logo-1.svg          ← логотипы в SVG (не PNG!)
├── logo-2.svg
├── photo-1.png         ← фото спикеров, квадратные, минимум 700×700px
├── photo-2.png
├── deco-1.svg          ← декоративные иллюстрации (если есть)
└── deco-2.svg
```

**Чеклист подготовки:**
- [ ] Все тексты написаны и согласованы (не «примерно», а финальные)
- [ ] Логотипы в формате SVG с белым или прозрачным фоном
- [ ] Фотографии обрезаны квадратно, лицо в верхней части
- [ ] Цвета согласованы (HEX-коды, не «бирюзовый»)
- [ ] Ссылки для QR-кодов известны
- [ ] Структура слайдов расписана (название каждого слайда и что на нём)

## Шаг 1. Опиши структуру одним сообщением

Отправь Claude одно большое сообщение со всей структурой сразу:

```
Создай HTML-презентацию 1920×1080, шрифт Manrope, 8 слайдов:

Цвета: фон тёмный #151515, акцент #00C8F0, светлый фон #fff

Слайд 1 — Обложка (тёмный фон)
- Логотип (файл logo.svg, белый, верхний левый угол)
- Заголовок: "Название / презентации" (140px, 400, белый)
- Декорация: файл deco.svg (справа, анимированные кольца)
- Футер: "КОМПАНИЯ  |  ГОРОД  |  МАЙ 2026"

Слайд 2 — Название раздела (тёмный фон)
- Заголовок: "Большой заголовок" (100px, 600)
- Подзаголовок: "Текст пояснения" (32px, 400)
- 2 колонки с текстом ниже

... и т.д. для каждого слайда
```

**Правило:** чем точнее описание — тем меньше правок. Указывай размер шрифта, цвет, позицию.

## Шаг 2. Проверяй по чеклисту после первого рендера

Перед тем как просить правки — пройдись по всем слайдам и составь **один список всех замечаний**. Не исправляй по одному — это замедляет работу.

Шаблон сообщения с правками:
```
Слайд 1: заголовок слишком маленький, нужно 140px
Слайд 3: убери фон у карточек, сделай их прозрачными
Слайд 5: нижние 2 карточки должны быть на всю ширину (grid-column 1/4 и 4/7)
Слайд 7: предлог "по" в карточке 2 висит — перенеси
Все слайды: футер кегль 14px uppercase
```

## Шаг 3. Типографика — задай один раз, не правь потом

Установи размеры сразу в первом сообщении:

| Элемент | Кегль | Вес | line-height |
|---|---|---|---|
| Главный заголовок (обложка) | 140px | 400 | 0.85 |
| Заголовок слайда | 100px | 600 | 0.9 |
| Подзаголовок | 50px | 500 | 1.2 |
| Заголовок карточки | 50px | 600 | 1.2 |
| Текст / тело | 32px | 400 | 1.3 |
| Имя спикера | 50px | 700 | 1.2 |
| Акцентный текст / формула | 100px | 600 | 1 |
| Лейбл uppercase | 22px | 700 | 1 |
| Футер | 14px | 500 | 1 |

## Шаг 4. Висячие предлоги — делай сразу

При написании текстов привязывай короткие предлоги к следующему слову через `&nbsp;`:

```
с&nbsp;клиентами
по&nbsp;задачам
в&nbsp;2025 году
и&nbsp;команда
от&nbsp;рассылки
до&nbsp;интерфейса
```

**Правило:** любой предлог или союз длиной 1–2 буквы в конце строки = `&nbsp;`.

## Шаг 5. SVG-иллюстрации — правила инлайнинга

### Логотипы
```html
<!-- Правильно: SVG inline, fill="currentColor" или fill="#fff" -->
<svg viewBox="0 0 231 50" width="231" height="50">
  <path d="..." fill="#fff"/>
</svg>

<!-- Неправильно: img-тег — нельзя управлять цветом через CSS -->
<img src="logo.svg">
```

### Декоративные иллюстрации с анимацией
```html
<div style="position:absolute;right:0;top:0;width:1200px;pointer-events:none">
  <svg viewBox="0 0 1053 803" fill="none" style="display:block;width:100%;height:auto">
    <!-- Статичные элементы — без классов -->
    <circle cx="700" cy="157" r="452" stroke="#02CBEF" stroke-width="6"/>

    <!-- Анимированные (пунктир) — class + --sd для скорости -->
    <circle cx="628" cy="87" r="347" stroke="#02CBEF" stroke-width="3"
            stroke-dasharray="14 14" class="spin-cw" style="--sd:90s"/>
    <circle cx="775" cy="236" r="563" stroke="#02CBEF" stroke-width="3"
            stroke-dasharray="14 14" class="spin-ccw" style="--sd:70s"/>
  </svg>
</div>
```

**Критично:** позиция через `cx`/`cy`, НЕ через `transform="matrix(...)"`.  
Если в оригинальном SVG есть `transform="matrix(-1 0 0 1 tx ty)"` — нужно пересчитать координаты:
```
new_cx = -cx_old + tx
new_cy =  cy_old + ty
```

## Шаг 6. Фотографии спикеров

```css
.photo-wrap {
  width: 350px;
  height: 350px;
  border-radius: 20px;
  overflow: hidden;
  position: relative;
}
.photo-wrap img {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
  object-position: top center; /* лицо не обрезается */
}
```

## Шаг 7. QR-коды

Не используй внешние API — они не работают без интернета. Используй qrcodejs:

```html
<script src="https://cdn.jsdelivr.net/npm/qrcodejs@1.0.0/qrcode.min.js"></script>

<div style="width:320px;height:320px;background:#fff;border-radius:16px;display:flex;align-items:center;justify-content:center">
  <div id="qr-1"></div>
</div>

<script>
new QRCode(document.getElementById('qr-1'), {
  text: 'https://example.com',
  width: 300, height: 300,
  colorDark: '#000', colorLight: '#fff',
  correctLevel: QRCode.CorrectLevel.M
});
</script>
```

## Шаг 8. Финальный чеклист перед показом

- [ ] Все тексты проверены, висячих предлогов нет
- [ ] Шрифт Manrope загружается (нужен интернет или предзагрузка)
- [ ] Фотографии не обрезают лица
- [ ] QR-коды рендерятся (проверь офлайн)
- [ ] Анимации работают (кольца вращаются)
- [ ] Навигация: стрелки, клик, свайп (мобильный)
- [ ] Точки навигации видны на светлых и тёмных слайдах
- [ ] PDF обновлён после последних правок
- [ ] Изменения запушены на GitHub/GitLab

## Шаг 9. Экспорт PDF

```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless=new \
  --disable-gpu \
  --no-pdf-header-footer \
  --print-to-pdf="presentation.pdf" \
  --paper-width=20 \
  --paper-height=11.25 \
  --no-margins \
  "file:///ПОЛНЫЙ/ПУТЬ/ДО/conf/index.html"
```
