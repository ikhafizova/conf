## Слайд 11 — Контакты / Спасибо

**Промт:**
> На 11 слайде нужно поставить изображение img_2 из папки и анимировать его так же как иллюстрация на первом слайде. Ссылки на сайты должны быть кликабельны. И давай вернем переключение слайдов — активный слайд полоска бирюзовая, неактивный серый кружок

**Промт 2:**
> анимация иллюстрации должна быть точно такой же как на слайде 1 в твоей презентации

**Проблема:** В `img_2.svg` элементы позиционированы через `transform="matrix(-1 0 0 1 tx ty)"`. CSS-анимация перезаписала матрицу → круги прыгали на неправильные позиции.

**Решение:** Вычислить реальные координаты центра каждого элемента применив матрицу:
```
Для matrix(-1 0 0 1 tx ty):
  new_cx = -cx_old + tx
  new_cy =  cy_old + ty
```
Убрать атрибуты `transform` из SVG, прописать вычисленные `cx`/`cy`.

**QR-коды:** `api.qrserver.com` не работает без интернета → заменить на `qrcodejs` CDN.

**Урок:** Перед инлайном SVG с анимацией проверить — нет ли у элементов `transform` атрибута. Если есть — вычислить финальные координаты и убрать атрибут.

## Навигационные точки

**Промт:**
> на белых слайдах кружочки в листалке должны быть светло-серые, сейчас их видно только на черных слайдах

**Исходный код:** `background: rgba(255,255,255,.28)` — не видно на белом фоне.

**Решение:**
```css
.dot {
  background: rgba(150,150,150,.45);
  box-shadow: 0 0 0 1.5px rgba(0,0,0,.12);
}
```

**Урок:** Точки навигации должны быть видны на любом фоне. `box-shadow` как обводка лучше чем менять цвет фона — работает и на светлом и на тёмном.

## PDF и публикация

**Промт:**
> мне нужна презентация в формате послайдового пдф без анимаций

**Решение:** `@media print` в CSS + Chrome headless:
```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless=new --print-to-pdf="presentation.pdf" \
  --paper-width=20 --paper-height=11.25 --no-margins \
  "file:///путь/index.html"
```
